# Diagnostics Documentation

## Overview
Diagnostics bridge the gap between internal compiler decisions and external observability (Tooling, Metrics, and Verification).

## `BackendSelectionDiagnostics`
Generated by `BackendSelectionFramework.select_backend()`, this object contains:
- `timestamp`: Float execution time.
- `candidate_backends`: A list of backends that were evaluated.
- `evaluations`: A dictionary mapping backend IDs to their detailed `BackendEvaluationReport`.
- `compatibility_reports`: A dictionary mapping backend IDs to their `CompatibilityReport`.
- `negotiations`: A dictionary mapping backend IDs to their `NegotiationDiagnostics`.
- `fallback_plan`: The full, generated `FallbackPlan`.
- `selected_backend`: The final chosen backend ID.

These objects are composed purely of immutable data structures (e.g. `MappingProxyType`, `tuple`) to ensure thread-safety when passed to background metric exporters.
