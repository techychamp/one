# Future CI Integration Notes

The Verification Framework is currently architectural. When integrating into full CI pipelines, observe these constraints:

- The verification suites must run on Apple Silicon hardware environments (M1/M2/M3/M4).
- Benchmark Verification tests should run in isolated runners to avoid noisy neighbour effects.
- Test suites are designed to provide metrics, not to optimize runtime execution. Do NOT introduce compilation arguments specifically for the CI environment.
- CI logs should parse the JSON output of the reports generated by `reporting.py`.
